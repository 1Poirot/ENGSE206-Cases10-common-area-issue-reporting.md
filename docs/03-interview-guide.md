## 1. Opening and Consent Script
> สวัสดีครับ/ค่ะ ทีมกำลังศึกษา workflow การขอใช้ห้องและอุปกรณ์เพื่อทำความเข้าใจปัญหา กฎ และข้อยกเว้นก่อนออกแบบระบบ การสนทนานี้ใช้เพื่อการเรียนในรายวิชา ENGSE206 เราจะไม่บันทึกข้อมูลส่วนบุคคลที่ไม่จำเป็น และคำตอบจะถูกสรุปเป็นหลักฐานตามบทบาท ไม่ระบุชื่อจริง ท่านสะดวกให้จดบันทึกหรือไม่ และมีประเด็นใดที่ไม่ควรถามหรือบันทึกหรือไม่

## 4. AI Interview Rehearsal

- Role/prompt used: “ขอให้คุณสวมบทบาทเป็นบุคคลที่เกี่ยวข้องกับการแจ้งและจัดการปัญหาในพื้นที่ส่วนกลางของอาคาร โดยสามารถเป็น [นักศึกษา / ผู้พบปัญหา, เจ้าหน้าที่อาคาร / เจ้าหน้าที่เทคนิค / แม่บ้าน, อาจารย์, ผู้ดูแลความปลอดภัย หรือผู้ดูแลระบบ] ให้ตอบตามประสบการณ์และขั้นตอนการทำงานจำลองของ Role นั้น”
- Session/evidence path: จำลองการสัมภาษณ์เพื่อทดสอบคำถามเกี่ยวกับ Workflow, Pain Point, Impact, Rule, Authority, Data, Exception และ Priority ของแต่ละ Role
- Simulation limitation: คำตอบของ AI เป็นเพียงการจำลองสถานการณ์ทั่วไป ไม่ใช่กฎระเบียบ นโยบาย หรือพฤติกรรมจริงของมหาวิทยาลัยหรือหน่วยงาน ดังนั้นต้องตรวจสอบข้อมูลกับผู้มีส่วนเกี่ยวข้องจริงก่อนนำไปใช้เป็น Requirement

## 5. Ready for Week 04

- Questions retained: คำถามเกี่ยวกับ Workflow การพบและแจ้งปัญหา ขั้นตอนการตรวจสอบและแก้ไข ปัญหาและผลกระทบ ข้อมูลที่จำเป็น กฎและอำนาจหน้าที่ กรณียกเว้น และการจัดลำดับความสำคัญ
- Questions removed/revised: ปรับคำถามที่อาจกว้างเกินไปหรือมีการตั้งสมมติฐานให้เป็นคำถาม Open, Probe และ Exception ที่เจาะจงมากขึ้น
- Information the simulated role did not know: AI ไม่สามารถยืนยันกฎ ระเบียบ ผู้มีอำนาจตัดสินใจ หรือขั้นตอนการจัดการเหตุการณ์จริงของมหาวิทยาลัยได้
- Open issues / verification needed: ต้องตรวจสอบกับผู้ใช้งานและเจ้าหน้าที่จริงเกี่ยวกับช่องทางการแจ้งปัญหาที่ใช้อยู่ ผู้รับผิดชอบในแต่ละขั้นตอน เกณฑ์การจัดลำดับความเร่งด่วน ขั้นตอนการส่งต่อปัญหา และข้อมูลที่จำเป็นต่อการแก้ไขปัญหา
